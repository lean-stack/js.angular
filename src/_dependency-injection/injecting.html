Injection

AngularJS kennt im Gegensatz zu anderen DI-Containern keine Property-Injection sondern lediglich Parameter-Injection (in Funktionen). Im Framework fest eingebaut ist die Möglichkeit Dienste (keine Provider) zu injezieren in

  • Module-Configblock (nur Provider und Konstanten)
  • Module-Runblock
  • Factories, Services
  • Controller
  • Direktiven

Die eigentliche Injection macht AngularJS im einfachsten Fall nur über den Namen: 

    function TodoController($scope,dayNameService) {
      $scope.todos = [];
    }

Der empfohlene Weg ist aber über eine Annotation. Der Guide nennt hier zwei Varianten:

  1. Inline-Array Notation 
    
    angular.module('name',[])
      .controller(['$scope','dayNameService', 
         function($scope,dayNameService) {
          $scope.todos = [];
         }
      ]);
  2. $inject-Property 
    
    function TodoController($scope,dayNameService) {
      $scope.todos = [];
    }
    TodoController.$inject = ['$scope','dayNameService'];

Dieser Annotations-Weg ist auch zwingend erforderlich, wenn die Scripte beim anschließenden Build minifiziert werden. Mit AngularJS 1.3 wurde eine ngStrictDi-Direktive eingeführt, die direkt am ngApp-Element angesetzt wird und eine Annotation erzwingt. Mit ng-annotate wurde ein Node-Tool entwickelt, dass nachträglich fehlende Annotationen einbaut.
