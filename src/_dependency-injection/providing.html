Providing Services

Was ist ein Service?
Ein Service in AngularJS kann alles mögliche sein. Grundsätzlich und ohne Ausnahme sind Services immer Singletons.
  • Ein einfacher Wert: Zahl, String, Boolescher Wert
  • Ein Array
  • Eine Funktion
  • Ein literales Objekt
  • Ein per Konstruktor erzeugtes Objekt
Wo wird ein Service definiert?
Die Bereitstellung im DI-Container (Dependency-Injection Container) geschieht über zwei Varianten:
  • $provide-Service in einem Module-Configblock
  • Helper- (Shortcut-) Methoden des Module-Interfaces
Siehe dazu die API-Dokumentation und das Beispiel im Konzepte-Teil.
Wie wird ein Service definiert?
AngularJS ist in der Dokumentation selbst etwas widersprüchlich, welche Arten von injectable components es gibt. Hier wird im Guide nur von Objects gesprochen, die mit einer Factory erzeugt werden, manchmal wird aber von services und values gesprochen. Ich fahre hier eine leicht unterschiedliche Betrachtungsweise (siehe obige Einteilung). Zur Erzeugung stehen insgesamt fünf Rezepte zur Verfügung.
  • Standard-Varianten
  • Provider
  • Factory
  • Value
  • Spezialfälle
  • Service-Klasse
  • Constant
Allgemeinfälle
Factory und Value sind dabei nur Vereinfachungen eines Providers.
Spezialfälle
Die Definition als Service-Klasse erlaubt nur über Konstruktoren definierte Objekte, der Constant-Service entspricht dem Value-Service, lässt sich aber nicht über einen Decorator dekorieren. (Fortgeschrittenes Thema, siehe API und zu einem Einstieg die StackOverflow-Frage). Ein schönes Einsatzbeispiel ist AJAX-Request Mocking.
